## 人脸情绪分析 MCP 工具

本工具用于通过本地摄像头捕获当前画面，检测其中的人脸，并基于本地情绪识别模型（如 DeepFace）分析每张人脸的主要情绪及整体情绪分布。

### 工具名称

- `analyze_face_emotion`

### 功能说明

- 打开本地摄像头（默认索引 0）采集当前画面；
- 自动检测画面中的人脸；
- 使用本地情绪识别模型（DeepFace）分析每张人脸的主要情绪及置信度；
- 汇总所有人脸的情绪分布，给出整体情绪概览；
- 支持两种返回模式：
  - 默认返回适合大模型理解的中文自然语言描述（格式化文本）；
  - 当 `return_raw=True` 时返回原始 JSON 结果列表。

### 适用场景

- 用户说出类似指令：
  - “分析摄像头前人脸情绪”
  - “看看我现在是什么心情”
  - “帮我识别一下我现在的表情”
  - “判断一下摄像头前面的人大概是什么情绪”

### 参数

工具使用以下输入参数（通过 MCP `tools/call` 的 `arguments` 传入）：

- `camera_index` (INTEGER，可选，默认 `0`)
  - 本地摄像头索引，`0` 通常为默认摄像头。
- `capture_timeout` (INTEGER，可选，默认 `3`，范围 `1–10`)
  - 采集画面的超时时间（秒），用于控制等待摄像头画面准备的最长时间。
- `return_raw` (BOOLEAN，可选，默认 `False`)
  - 返回模式控制：
    - `False`（默认）：返回格式化的中文自然语言描述；
    - `True`：返回原始 JSON 结果列表，包含每张人脸的详细情绪数据。

### 返回结果

**默认模式（`return_raw=False`）：**
- 返回格式化的中文自然语言描述，包含：
  - 检测到的人脸数量；
  - 每张人脸的主要情绪及大致置信度（百分比）；
  - 聚合后的整体情绪分布（例如"高兴约 60%，中性约 30%"等）。

**原始数据模式（`return_raw=True`）：**
- 返回 JSON 字符串格式的原始结果列表，每个元素包含：
  - `dominant_emotion`：主要情绪（字符串）
  - `confidence`：置信度（浮点数）
  - `all_emotions`：所有情绪的详细分布（字典，键为情绪名称，值为置信度）

**错误情况：**
- 未检测到人脸或分析失败时，会返回清晰的中文提示信息，例如：
  - "未在画面中检测到人脸，或情绪分析结果为空。"
  - "人脸情绪分析未完成：无法打开摄像头……"

### 错误处理

- 常见可预期错误（如未安装依赖、无法打开摄像头、无检测到人脸）会在工具内部捕获，并以友好中文文本形式返回。
- 内部未预期异常会记录日志，并由 MCP 框架包装为 `isError: true` 的响应。

### 依赖

要正常使用本工具，需要在运行环境中安装：

```bash
pip install opencv-python deepface
```

> 注意：DeepFace 可能依赖额外的后端（如 TensorFlow/PyTorch）和预训练模型，首次运行时可能会下载模型文件，请确保网络和磁盘空间可用。
